# 朝当番プログラム（朝掃除スケジュール生成）の詳細説明

各階（1階〜5階）の朝掃除のシフト表を自動生成し、寮生データを管理するプログラムです。
「階ごとに異なる複雑な回転規則（ターン）」や「他階からのサポートスタッフシステム」**を網羅しつつ、ユーザーの使いやすさと将来のルール変更が起こった際も対応できるようにプログラムしました．

---

##  ユーザーに優しい入力設計

各階によって名簿の回転方向（ターン）が「正順」だったり「逆順」だったりするため、本来ならデータの入力順序も階ごとに変えなければならず、人間が行うと混乱してしまいます。

本プログラムでは、その複雑な処理を内部の `switch` 文とループ（`start2` や `end2` の自動切り替え）で完全に吸収しています。
* **特徴**: ユーザーは、**作成する階のターンがどれであっても、手元にある先月のシフト表の「左から順番に」部屋番号を入力していくだけで100%正しいデータが取り込まれます。**
システムが裏側で自動的にその階のターンに合わせた順序に翻訳して処理するため、ユーザーが入力時に迷うことは一切ありません。

---

## ルールの変更への即時対応可能

将来的に寮の規則や運用の仕組みが変更された場合でも、**ソースコードの深いロジック（関数の中身）を書き換えることなく、プログラム先頭の「ルール設定エリア」のマクロ（`#define`）を書き換えるだけで即座に対応できる**設計にしています。

* **掃除場所の数の変更**: `#define AREA_COUNT 4` の数値を変更するだけで、すべての配列やループ処理が自動的に連動します。
* **各階の回転方向（ターン）の変更**: `floor1_turn` 〜 `floor5_turn` の数値を書き換えるだけで、各階のローテーション方向や画面への出力方向を個別にいつでもカスタマイズできます。
* **免除条件（学年・月）の変更**: 免除学年の基準や、免除を適用する月（現在は全月に対象）の条件式も、マクロを変更するだけでシステム全体に一発で反映されます。

## ロジック1：朝掃除の超複雑な `search` 関数（当番探索アルゴリズム）

朝当番の `search` 関数は、通常のスキップ判定に加え、**「サポートスタッフ（他階の住人）の誤認識防止」**と**「出張中の元フロアでの免除権消費」**を同時に行うため、3段階のフィルターで組んでいます．

* **第1フィルター：絶対対象外のスキップ**
  `features < 0`（役員など）や `temp_exempt == 1`（学年免除）の人は、一切の例外なくスルーします。
* **第2フィルター：他階へ出張中の住人の処理**
  本来の部屋がある階（部屋番号の百の位）と、今現在割り当てられている掃除階（`data.floor`）が一致しない人（＝他階へ朝掃除サポートに行っている人）を検知します。
  その人の「本来の所属階」の当番表を作っている最中に順番が回ってきた場合、**持っている免除権（`exemption`）を裏側で1つ減らす（消費させる）ことで、出張中であっても不公平なく順番を消化させる**処理を行います。
* **第3フィルター：その階の本来の住人の処理**
  部屋番号がその階の正規の住人であることを確認した上で、免除権があれば消費して回避させ、免除権がなければその日の当番（`shift_table`）に確定させます。このとき、サポートスタッフ（一時的にこの階に所属させられている他階の人）が一般枠に間違えて巻き込まれないよう、厳密な部屋番号チェックを行っています。

当番が決まった後は、その階固有のターン方向（正順なら `target_idx++`、逆順なら `target_idx--`）へとルーレットの針を正しく進めます。

## ロジック2：「朝掃除サポートスタッフ」の出張と報酬システム

他階の住人を「サポートスタッフ」として一時的に招き入れ、シフトに組み込む高度な動的ロジックです。必ず，サポートスタッフが入る階からシフトを作るようにしてください．

### 1. 一時的な「所属階」の書き換え
サポートスタッフの部屋番号が入力されると、プログラムは対象者の構造体データ（`data[j].floor`）を、一時的にその当番表を作っている対象階へと書き換えます。これにより、通常はその階の住人しかヒットしない `search()` 関数の網に、サポートスタッフが自然に引っかかるようになります。

### 2. シフト配列（斜めスライド）への組み込み
朝掃除のシフトは、前日のメンバーを右へ1つずつずらす「斜めスライド」で作成されます。
* サポートスタッフがいる場合、1日目の作成 (`make1`) で一番左の枠（`[0]` の位置）をスタッフ用として確保するため、一般のルーレットのスタート位置を `min = 1` に底上げします。
* 2日目以降、スタッフがローテーションを1周して再び最前列に戻ってくる条件（1日目の配置と重なる、または他階の住人が一般枠に混ざりそうになる瞬間）を検知すると、`turn_second` 関数が自動でスタッフを `[0]` 枠へ再配置（リピート処理）します。

### 3. 免除権の自動付与
すべてのシフトを組み終えたあと、プログラムは完成した当番表の最前列（`shift_table[j][0]`）を自動でスキャンします。検出された回数分免除権を与えます．

## 主な関数一覧と役割

* `main()`: データの読み書きに加え、先月シフトの入力方向を階のターンに合わせて自動で逆転制御する役割を持つ。
* `search()`: 3段階のフィルターを用いて、出張中スタッフの免除権消費や一般住人の当番決定を、ターンの回転方向に合わせて行う最重要関数。
* `make1()`: 前月最後の当番の続きから、今月の「第1日目」の初期シフトを組み立てる。サポートスタッフ用の枠確保（底上げ）もここで行う。
* `turn_second()`: 2日目以降のシフトを斜めスライドで生成し、サポートスタッフが2周目に突入するタイミングを検知して最前列へ自動リピート配置する。
