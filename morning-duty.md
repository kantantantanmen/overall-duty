# 朝当番プログラム（朝掃除スケジュール生成）の詳細説明

各階（1階〜5階）の朝掃除のシフト表を自動生成し、寮生データを管理するプログラムです。
このプログラムは、**「階ごとに異なる複雑な回転規則（ターン）」**や**「他階からのサポートスタッフシステム」**を網羅しつつ、ユーザーの使いやすさと将来のルール変更への強さを両立しています。

---

## ⚙️ 高いメンテナンス性（ルールの変更への即時対応）

将来的に寮の規則や運用の仕組みが変更された場合でも、**ソースコードの深いロジック（関数の中身）を書き換えることなく、プログラム先頭の「ルール設定エリア」のマクロ（`#define`）を書き換えるだけで即座に対応できる**設計にしています。

* **掃除場所の数の変更**: `#define AREA_COUNT 4` の数値を変更するだけで、すべての配列やループ処理が自動的に連動します。
* **各階の回転方向（ターン）の変更**: `floor1_turn` 〜 `floor5_turn` の数値を書き換えるだけで、各階のローテーション方向や画面への出力方向を個別にいつでもカスタマイズできます。
* **免除条件（学年・月）の変更**: 免除学年の基準や、免除を適用する月（現在は全月に対象）の条件式も、マクロを変更するだけでシステム全体に一発で反映されます。

---

## 📥 ユーザーに優しい入力設計（先月データの入力ルール）

各階によって名簿の回転方向（ターン）が「正順」だったり「逆順」だったりするため、本来ならデータの入力順序も階ごとに変えなければならず、人間がやると混乱してしまいます。

本プログラムでは、その複雑な処理を内部の `switch` 文とループ（`start2` や `end2` の自動切り替え）で完全に吸収しています。
* **特徴**: ユーザーは、**作成する階のターンがどれであっても、手元にある先月のシフト表の「左から順番に」部屋番号を入力していくだけで100%正しいデータが取り込まれます。**
システムが裏側で自動的にその階のターンに合わせた順序に翻訳して処理するため、ユーザーが入力時に迷うことは一切ありません。

---

## 🔄 核心ロジック1：階ごとの「ターン（回転・表示方向）」制御

寮の運用に合わせ、階ごとに名簿を「正順」で回すか「逆順」で回すか、さらに画面に出力する方向までを自動制御しています。

* `floor1_turn 4` / `floor2_turn 4`（逆順・右から左へ出力）
* `floor3_turn 2`（逆順・左から右へ出力）
* `floor4_turn 3`（正順・右から左へ出力）
* `floor5_turn 2`（逆順・左から右へ出力）

### ターンがプログラムに与える影響
変数 `turn_direction` の値（1〜4）によって、以下の処理が自動で切り替わります。
* **探索方向の切り替え (`search` / `make1` 関数)**: 当番のルーレットを回す際、正順なら名簿を順に進め（`target_idx++`）、逆順なら逆方向に進めます（`target_idx--`）。
* **表示方向の切り替え (`print` / `turn_second` 関数)**: 1日分のシフト（4箇所分）を画面に表示する際、左から右へ出力するか、右から左へ出力するかをループの増減（`j++` または `j--`）で自動制御します。

---

## 🧹 核心ロジック2：「朝掃除サポートスタッフ」の出張と報酬システム

人手が足りない階に対して、他の階の住人を「サポートスタッフ」として一時的に招き入れ、シフトに組み込む高度な動的ロジックです。

### 1. 一時的な「所属階（出張）」の書き換え
サポートスタッフの部屋番号が入力されると、プログラムは対象者の構造体データ（`data[j].floor`）を、**一時的にその当番表を作っている対象階（`target_floor`）へと書き換えます。**
これにより、通常はその階の住人しかヒットしない `search()` 関数の網に、サポートスタッフが自然に引っかかるようになります。

### 2. シフト配列（斜めスライド）への組み込み
朝掃除のシフトは、前日のメンバーを右へ1つずつずらす「斜めスライド」で作成されます。
* サポートスタッフがいる場合、1日目の作成 (`make1`) で一番左の枠（`[0]` の部屋）をスタッフ用に固定で空けておき（`min=1` の底上げ処理）、そこへスタッフを配置します。
* 2日目以降、スタッフがローテーションを1周して再び最前列に戻ってくる条件を検知すると、`turn_second` 関数が自動でスタッフを `[0]` 枠へ再配置（リピート処理）します。

### 3. がんばった人への「免除権（報酬）」の自動付与
シフト表を作り終えたあと、プログラムは生成されたスケジュール全体を自動スキャンします。
```c
if (shift_table[j][0] == data[staff[i]].room_num) {
    data[staff[i]].exemption++; // 1つでも掃除に入ったら免除権ゲット！
}
