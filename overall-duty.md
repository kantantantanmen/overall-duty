# overall-duty（全体当番スケジュール自動生成システム）

寮生（定員108名）の「ゴミ当番」「風呂当番」「寮務当番」の3つのシフトスケジュールを自動生成し、寮生データ（CSV形式）の更新・管理までを一括で行うC言語プログラムです。

それぞれの当番が持つ独自のルールや各種「免除権」、新学期の入れ替え処理などに柔軟に対応しています。

---

## ⚙️ 高いメンテナンス性（ルールの変更への即時対応）

将来的に寮の規則（ルール）が変更された場合でも、**ソースコードの深いロジックを書き換えることなく、プログラム先頭の「ルール設定エリア」のマクロ（`#define`）を変更するだけで即座に対応できる**設計にしています。

* **免除対象学年の変更**: 
  4年制・6年制それぞれの免除学年（例: 4年生から ➡️ 3年生から、など）を数値を1箇所変えるだけでシステム全体に反映できます。
* **免除対象月の変更・追加**: 
  現在は1月〜3月が免除となっていますが、マクロ内の条件式（`RULE_EXEMPT_MONTH`）に `|| (m) == 12` のようにつなげて書くだけで、免除する月を簡単にカスタマイズ・増減できます。

---

## 🛠 3つの当番システムと固有ルール

本プログラムは、当番ごとの異なる運用ルールを完全にシミュレートしています。

### 1. ゴミ当番 (`garbage`)
* **ルール**: 毎週「月曜日」と「木曜日」の週2回実施。
* **特徴**: 開始曜日（月曜または木曜）をユーザーが指定すると、自動でカレンダーの周期（3日後・4日後）を計算し、前月の最終当番人の次の人から順番に割り当てます。

### 2. 風呂当番 (`bath`)
* **ルール**: 毎日実施。
* **特徴**: 前月の最終当番人の次の人から、前半・後半の指定日数分、順番に隙間なく均等に割り当てます。

### 3. 寮務当番 (`dormitory`)
* **ルール**: 毎日実施。ただし「土日」は特殊ルールが適用されます。
* **特徴【夜間生自動配置】**: 学校の都合で平日に動けない夜間生（`FEATURE_NIGHT`）を検知すると、自動的に直近の「土曜日」または「日曜日」の枠を探して**スライド配置（予約配置）**を行います。これにより、通常生のローテーションを崩さずに休日当番を消化させます。

## 🌟 システムの主な特徴と管理ロジック

### ① 特徴フラグ（`features`）による状態管理
寮生一人ひとりの状態を数値（フラグ）で管理し、割り当て対象から自動で除外・選別します。
* `0`：通常寮生（平日・休日問わず順番に割り当て）
* `1`：夜間生（平日の寮務当番をスキップし、土日にスライド）
* `10`：夜間生関連（先月当番に入った、または免除権を使用したため今月はスキップ）
* 負の数（`-1`〜`-11`）：各種免除者（空き部屋、休学者、前期・後期役員など。当番から完全除外）

### ② 4月・5月の新旧メンバー入れ替え処理
年度替わりの複雑なデータ更新を自動化しています。
* `month4mae()`：卒業生の部屋を「空き部屋」に初期化し、在校生の学年を自動で1つ上げます。
* `month4ato()`：空いた部屋に新1年生を自動で入居（データ初期化）させます。
* `month511()`：任期を終えた旧役員の免除権を剥奪し、通常生や夜間生へ再設定します。

### ③ 堅牢なエラーハンドリング (`get_valid_input`)
ユーザーがメニュー選択や部屋番号の入力で、誤って「文字」を入力したり存在しない部屋番号を打ったりした場合でも、プログラムが強制終了（無限ループ）せず、正しい入力を促すバリデーション機能を搭載しています。

## 💾 データ構造とファイルI/O

入寮生データは `roomdata` 構造体で管理され、外部ファイルからカンマ区切り（CSV形式）で読み込み（`r`）、生成完了後に最新の状態に更新して上書き保存（`w`）されます。

```c
// 1人あたり8つのパラメーターで管理
typedef struct DATA{
    int room_num;    // 部屋番号
    int grade;       // 学年
    int floor;       // 階数
    int exemption;   // 免除権総数
    int exemption1;  // ゴミ免除
    int exemption2;  // 風呂免除
    int exemption3;  // 寮務免除
    int features;    // 特徴フラグ（夜間生・役員など）
} roomdata;
